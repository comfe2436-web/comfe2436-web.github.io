# Заметки по интервью

## Интервью 1: менеджер продаж

**Боли**

- заявки приходят из разных каналов;
- часть данных приходится уточнять после начала обработки;
- статусы ведутся вручную и по-разному у разных менеджеров;
- сложно понять, кто сейчас отвечает за заявку.

**Потребность пользователя**

Менеджеру нужна единая форма заявки с обязательными полями, подсказками и автоматическим назначением срока.

## Интервью 2: руководитель отдела продаж

**Боли**

- нет ежедневной картины по просроченным заявкам;
- отчет собирается вручную;
- руководитель узнает о проблемах слишком поздно;
- нет единой логики приоритетов.

**Потребность пользователя**

Руководителю нужен отчет по статусам, SLA, ответственным и заявкам высокого приоритета.

## Интервью 3: бухгалтерия и склад

**Боли**

- в заявках не хватает данных для оплаты или отгрузки;
- приходится возвращать заявку менеджеру;
- нет прозрачного статуса ожидания уточнений.

**Потребность пользователя**

Смежным подразделениям нужны обязательные поля, история статусов и понятный маршрут доработки заявки.
