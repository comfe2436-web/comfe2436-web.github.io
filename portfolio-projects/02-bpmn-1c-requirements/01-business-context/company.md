# Контекст компании

## Компания

ООО "ТехСнаб" - компания по продаже промышленного оборудования для производственных предприятий.

| Параметр | Значение |
|---|---|
| Сфера | Продажа промышленного оборудования |
| Количество сотрудников | 65 |
| Основная система | 1С |
| Средний объем заявок | около 200 в день |
| Основные подразделения | продажи, бухгалтерия, склад |

## Текущая ситуация

Клиентские заявки поступают через почту, мессенджеры и устные обращения. Менеджеры вручную переносят данные в 1С, уточняют недостающую информацию, согласуют нестандартные заявки с руководителем и передают задачи в работу смежным подразделениям.

## Бизнес-проблема

Процесс зависит от ручных действий и личной дисциплины участников. Из-за этого возникают задержки, неполные данные, разные статусы, потерянные заявки и слабый контроль сроков.

## Цель проекта

Подготовить аналитический пакет для доработки 1С, который поможет стандартизировать обработку заявок, снизить долю ручных операций и дать руководителю прозрачный контроль статусов и SLA.

## Ожидаемый бизнес-эффект

- единый канал фиксации заявок в 1С;
- снижение количества заявок с неполными данными;
- прозрачные статусы и ответственные;
- контроль SLA и просрочек;
- отчетность для руководителя без ручной консолидации.
